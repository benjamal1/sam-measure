<objective>
Redesign the interactive/batch surface of the pipeline around the user's now-cleaned real folder structure (`Condition/Batch N DATE/D# DATE/file.JPG`, one `ruler_MM-DD-YY.JPG` per day, some photos containing multiple threads). Five changes, confirmed with the user directly:

1. `segment_export` walks the entire nested tree in one run — no restarting the CLI per photo.
2. Multi-thread-per-photo labeling: after accepting a mask, user chooses to label another thread on the SAME photo or advance to the next photo. Condition + thread number are typed manually at label time (never parsed from the path — EXPT-01 revised: manual entry replaces path-derived metadata as the primary source, path parsing becomes a display hint at most, not authoritative).
3. Ruler calibration matched by date; a session date with no same-date ruler falls back to the most recent EARLIER-dated ruler within the SAME batch (never cross-batch, per user confirmation — different batches may have different camera/lighting setup).
4. `area_px`/`area_mm2` and `mad_px`/`mad_mm` (median absolute deviation — more robust to SAM2's jagged mask-boundary noise than stdev, per the Phase-2 VALIDATION finding of ~30% stdev overshoot) are added to measurements.csv and final.csv. `avg_diameter_px`/`stdev_px` and `EXACT_R_SCRIPT_COLUMNS` order are UNCHANGED — R script keeps working unmodified; new columns are additive, appended after.
5. Outlier flagging (pulls v2 QOL-02 forward): a thread whose `area_px` or `avg_diameter_px` is an IQR-based outlier within its own (date,batch,condition) sibling group gets a `flag` column set, with the QC overlay path surfaced in the row so the user can jump straight to the image for a mislabel check.

Masks/QC overlays already export to `data/masks/` and `data/qc/<stem>_overlay.png` (existing EXPT-03, unchanged) — this plan does not touch that export path, only what feeds INTO it (multi-mask loop) and what's computed FROM the masks afterward (measurement + calibration + outlier columns).
</objective>
