Harden the Stage-1 export path and the end-to-end orchestrator for real batch runs: make re-export idempotent (skip already-exported photos, `--force` to override) and make every run leave a JSON audit manifest. Implements EXPT-04 (D2-01: skip-if-output-exists keyed by the canonical mask filename) and CSV-05 (D2-05: per-run JSON manifest recording inputs, outputs-written-or-skipped, calibration factor per thread, and errors).

Every existing module is reused unchanged except the two entry points that loop over a batch (segment_export.py, run_pipeline.py). MVP vertical-slice framing does not apply to a hardening phase; success is "the batch run is safe to re-run and leaves an audit trail," proven against the real 08-04-25 folder.

Purpose: a batch of 24+ photos must be re-runnable without duplicating or clobbering masks, and every run must be auditable after the fact.
Output: `src/pipeline/manifest.py`, idempotent `export_folder()`, orchestrator manifest writing, and real-data + unit tests.
</objective>
