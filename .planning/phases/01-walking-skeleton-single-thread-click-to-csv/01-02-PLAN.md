<objective>
Deliver the fully-automated, headless measurement stage (D-03): `measure_mask()` turns a binary mask
into `area_px`, `avg_diameter_px`, and `stdev_px` via skimage `skeletonize` + scipy
`distance_transform_edt` with boundary cleanup and endpoint trimming (MEAS-01, MEAS-02) — the exact
skeleton+distance-transform method locked by D-03, with the true perpendicular ray-cast approach
deliberately deferred to v2 per D-04. A thin Stage-2 CLI runs `measure_mask` over a folder of exported
masks and writes `data/csv/measurements.csv` in the frozen schema (SKELETON.md).

Purpose: This is the "pixel math" half of the pipeline's correctness. It must be independently
re-runnable and unit-testable against fixed mask fixtures (ARCHITECTURE Pattern 1) — never coupled to
SAM2/GUI — so a diameter-averaging fix later needs no re-segmentation. Phase 2's MEAS-03 will validate
its numbers against ImageJ ground truth; Phase 1 only requires the method be correct-in-shape and
plausible on real masks.

Output: `src/measure/measure_masks.py` (pure function + CLI) and its test suite. After this plan a user
can run the measurement stage over a mask folder and get `measurements.csv` (AC-006, AC-009).
</objective>
