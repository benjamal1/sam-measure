<objective>
Establish the walking-skeleton scaffold (directory layout contract from SKELETON.md, pytest
configuration, frozen requirements, gitignored `data/`) and deliver the first real pure module:
`naming.py`, the shared folder-path metadata parser (EXPT-01, D-07) that treats the day/measurement
folder date as the canonical CSV `Date` (D-09).

**Phase user story:** As a thread-compaction researcher, I want to click once to segment a thread
photo, correct the mask if SAM2 grabs the needle, and run the pipeline against a ruler photo, so that
I get a correct CSV row in the exact column format my existing R script consumes — without manual
ImageJ edge-tracing. (This plan lays the scaffold + naming slice; later plans add measure, calibrate,
segment, and the end-to-end join.)

Purpose: Every downstream stage derives its identifying CSV columns by parsing the mask filename that
`canonical_stem()` produces, so this module is the shared contract both Wave-2 plans (measure, join)
and Wave-3 (segment export) depend on. Getting the Date-field disambiguation right here (D-09) prevents
a silent, systematic data-correctness bug in every CSV row.

Output: importable `src/` package layout, `pyproject.toml`/`requirements.txt`/`.gitignore`, a
comprehensive `tests/conftest.py` (finalized here — Wave-2 plans must NOT modify it), a `run_pipeline.py`
stub, and a fully TDD'd `naming.py`.
</objective>
